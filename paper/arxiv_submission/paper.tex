\section{Summary}\label{summary}

\texttt{HydraR} is a lightweight, state-managed framework designed to
orchestrate complex ``agentic'' workflows that are autonomous, multi-step
processes driven by Large Language Models (LLMs) natively within R.
While mainstream orchestration tools such as LangChain
(\citeproc{ref-langchain}{Chase, 2022}) and CrewAI
(\citeproc{ref-crewai}{Moura, 2023}) are predominantly Python-based,
\texttt{HydraR} fulfills the R community's critical requirements for
rigorous reproducibility, auditability, and seamless integration with
established statistical pipelines.

Researchers can architect workflows as Directed Acyclic Graphs (DAGs) or iterative state machines, where each node represents an LLM-prompted task, a deterministic R function, or an autonomous auditor. \texttt{HydraR} provides a provider-agnostic interface that supports cloud APIs (Gemini, Claude, OpenAI, GitHub Copilot) and local deployments via Ollama. Workflow topologies are defined via declarative YAML manifests that embed \texttt{Mermaid.js} syntax for visualization and conditional edge logic (e.g., "if fail, go to Node B") for reactive routing. To ensure reliability, a built-in validation engine performs deep static analysis of these manifests—executing syntactic parse checks, and verifying topological synchronization. While \texttt{HydraR} permits cyclic dependencies to facilitate agentic self-correction, it enforces runtime safety through a configurable execution "fuse" (limiting transitions to 25 steps by default) and persistent state checkpointing.

\section{Statement of Need}\label{statement-of-need}

The contemporary R ecosystem lacks a unified framework providing both high-level agentic orchestration and robust, low-level filesystem safety. While existing packages, such as \texttt{ellmer} (\citeproc{ref-ellmer}{Wickham et al., 2025}), focus primarily on conversational interfaces and \texttt{mall} (\citeproc{ref-mall}{Couch, 2024}) excels at mapping LLM inferences over structured data, neither is equipped to manage the complex state persistence and isolated execution environments necessitated by long-running, multi-agent collaborations.

\texttt{HydraR} addresses three fundamental challenges in this domain:
\begin{enumerate}
    \item \textbf{Auditability}: It implements a centralized \texttt{AgentState} with persistent checkpointing (via DuckDB \citeproc{ref-duckdb}{Raasveldt \& Mühleisen, 2019}), yielding a complete, verifiable audit trail of all LLM interactions and state mutations.
    \item \textbf{Filesystem Safety}: Leveraging Git worktrees, \texttt{HydraR} isolates parallel agent executions into temporary branches, thereby virtually eliminating data corruption and race conditions during concurrent operations.
    \item \textbf{Portable Logic Topologies}: By defining workflows via \texttt{Mermaid.js} syntax and \texttt{YAML} manifests, \texttt{HydraR} entirely decouples orchestration intent from the underlying execution engine. This declarative schema functions as an ``OpenAPI for Agents'' — an intermediate, engine-agnostic representation that not only ensures robust auditability in R but also serves as a highly portable compilation target for broader, code-first frameworks such as LangGraph or Google's Agent Development Toolkit.
\end{enumerate}


\section{State of the Field}\label{state-of-the-field}

Although \texttt{reticulate} (\citeproc{ref-reticulate}{Ushey et al., 2024}) enables R users to interface with Python frameworks like LangChain, it introduces a mismatch between the two languages’ data models when serializing complex R objects across language boundaries. \texttt{HydraR} avoids this by building natively on the \texttt{R6} object-oriented system (\citeproc{ref-R6}{Chang, 2021}), preserving the fidelity of statistical objects while maintaining direct compatibility with R’s integrated debugging toolchains. Furthermore, unlike \texttt{ellmer} (\citeproc{ref-ellmer}{Wickham et al., 2025}) (which offers chat-centric workflows without DAG orchestration), and \texttt{gptstudio} (\citeproc{ref-gptstudio}{Nivard \& Wade, 2024}) (which serves as an IDE-centric coding assistant), \texttt{HydraR} is specifically engineered to manage the full life-cycle of autonomous, parallel, multi-agent workflows with robust state retention.

\section{Software Design}\label{software-design}

\texttt{HydraR} employs a highly modular \texttt{R6} architecture
spanning several key components: 

* \texttt{AgentLLMNode} /
\texttt{AgentLogicNode}: Encapsulate LLM-driven prompts and
deterministic pure-R logic steps, respectively. They utilize pluggable
\texttt{AgentDriver} backends for provider-agnostic model communication.

* \texttt{AgentDAG}: The core orchestration engine that manages the node
dependencies, validates network topology (via \texttt{igraph}
(\citeproc{ref-igraph}{Csárdi \& Nepusz, 2006})), and executes logic in
parallel (via \texttt{furrr} (\citeproc{ref-furrr}{Vaughan \& Reznik,
2022})). It features an integrated compiler-like Validation Engine that
treats the YAML/Mermaid manifest as a strict Domain Specific Language (DSL).
This engine cross-references Mermaid topologies with \texttt{YAML} resource
definitions and lints embedded R logic for regulatory compliance (e.g.,
prohibiting imperative loops), effectively positioning \texttt{HydraR} as a
rigorous reference implementation for executing portable agentic protocols.

* \texttt{AgentState}: A
centralized state repository employing versioned history and custom
reducers to coordinate multi-agent outputs systematically. -
\texttt{WorktreeManager}: An engine that provisions isolated Git
worktrees for parallel execution tasks, paired with an extensible
\texttt{ConflictResolver} that performs automated semantic resolution
alongside human-in-the-loop task reconciliation.

\subsection{Multimodal Travel Itinerary
Planner}\label{multimodal-travel-itinerary-planner}

In this example, an LLM \texttt{Planner} agent generates a draft
itinerary, which is then audited by a deterministic \texttt{Validator}
against hard constraints. Successfully validated plans proceed to a
\emph{multimodal image generation} phase where \textbf{Gemini 3.1 Flash}
(\citeproc{ref-google2025gemini}{Google, 2025}) creates visual assets
based on the itinerary's locales. Finally, a logic node renders a
bespoke, CSS-styled HTML pamphlet. Crucially, \textbf{conditional
looping} between the planner and validator enforces an iterative
refinement cycle until all strict parameters are met

\subsection{Parallel Sorting Algorithm
Comparison}\label{parallel-sorting-algorithm-comparison}

This benchmarking example tasks three distinct LLM agents with
simultaneously implementing different sorting algorithms.
\texttt{HydraR} executes each task within an isolated
\texttt{Git\ worktree} to aggressively uncouple filesystem side-effects
and prevent merge conflicts. Following execution, a Merge Harmonizer
node systematically reconciles the independent branches back to the
main state, preceding a terminal logic node that empirically benchmarks
the aggregated algorithms across five continuous trials
(\autoref{fig:sorting}).

\subsection{Fault-Tolerant Pipelines with DuckDB
Persistence}\label{fault-tolerant-pipelines-with-duckdb-persistence}

To mitigate transient failures inherent to long-running scientific
workflows, \texttt{HydraR} integrates advanced checkpoint-and-resume
functionality powered by DuckDB (\citeproc{ref-duckdb}{Raasveldt \&
Mühleisen, 2019}). Whenever a pipeline pauses mid-execution, it
automatically persists its complete \texttt{AgentState}. Upon restart,
any operator-applied programmatic interventions are seamlessly merged,
actively preventing redundant evaluations by allowing execution to jump
directly back to the paused node.

\section{Research Impact Statement}\label{research-impact-statement}

\texttt{HydraR} is actively deployed at the \textbf{Australian Proteome
Analysis Facility (APAF)} to architect and automate resilient
bioinformatics tooling. Its declarative YAML schemas empower
non-programming domain experts to routinely govern advanced LLM logic
pipelines without any R coding proficiency, thereby democratizing the
scale and reproducibility of model-assisted scientific research.
Furthermore, by championing a ``State-Machine-as-a-Document'' standard
over proprietary, code-bound execution graphs, \texttt{HydraR} pioneers
a universal, engine-agnostic blueprint for multi-agent workflows. This
standardized format bridges the visualization gap between domain researchers
and software engineers, ensuring complex protocols remain infinitely
auditable, version-controllable, and portable across disparate AI ecosystems.

\section{AI Usage Disclosure}\label{ai-usage-disclosure}

Transparency and accountability are core tenets of the \texttt{HydraR}
paradigm. A substantial portion of the implementation was co-developed
utilizing \textbf{Antigravity}, an agentic AI coding assistant,
supervised by a rigorous ``human-in-the-loop'' review protocol.
Specifically, the original authors independently conceptualized the
underlying system architecture, while the AI generated specialized
logic, unit tests, and software documentation with every resultant
commit sequentially tested and manually verified.
