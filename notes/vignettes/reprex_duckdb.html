DuckDB Restart Reprex: Automated Workflow Persistence

Introduction

This vignette demonstrates a Minimal Reproducible Example (Reprex) for the HydraR core feature: Automated State Persistence & Restart.

In high-complexity multi-agent workflows, code can fail due to environmental factors, API timeouts, or logic errors. HydraR allows you to: 1. Checkpoint every step of a DAG to a DuckDB database. 2. Pause execution when a step encounters a problem. 3. Resume exactly from the paused node after the state has been “fixed.”

Setup

Load the library and define a temporary DuckDB path.

library(HydraR)
db_path <- tempfile(fileext = ".duckdb")

1. Defining the Logic

We register three logic functions globally. Note how check_fixed inspects the state for a fixed flag — if it’s not TRUE, the node pauses execution instead of crashing.

register_logic("init_proc", function(state) {
  list(status = "success", output = "Initialization complete.")
})

register_logic("check_fixed", function(state) {
  if (!isTRUE(state$get("fixed"))) {
    # Return "pause" status — HydraR saves state and stops cleanly
    return(list(status = "pause", output = "Waiting for manual fix."))
  }
  list(status = "success", output = "System recovered!")
})

register_logic("finalize_proc", function(state) {
  list(status = "success", output = "All steps finished.")
})

2. Building the DAG Inline

We define a simple 3-step pipeline using Mermaid syntax.

mermaid_src <- '
graph TD
  Step1["Initialization | type=logic | logic_id=init_proc"]
  Step2["Risky Logic | type=logic | logic_id=check_fixed"]
  Step3["Conclusion | type=logic | logic_id=finalize_proc"]
  Step1 --> Step2
  Step2 --> Step3
'

dag <- mermaid_to_dag(mermaid_src)
dag$set_start_node("Step1")

3. Initial Run (The Pause)

We run the DAG with fixed = FALSE. Step2 will pause the pipeline and the checkpointer saves the state to DuckDB.

# Configure DuckDB Persistence
saver <- DuckDBSaver$new(db_path = db_path)
tid <- "reprex-session-001"

# Run 1: Expected to pause at Step2
res1 <- dag$run(
  thread_id = tid,
  checkpointer = saver,
  initial_state = list(fixed = FALSE, counter = 0)
)
#> Graph compiled successfully.
#> [Linear] Running Node: Step1
#>    [Step1] Executing R logic...
#> [Linear] Running Node: Step2
#>    [Step2] Executing R logic...

cat("Execution Status:", res1$status, "\n")
#> Execution Status: paused
cat("Paused at:", res1$paused_at, "\n")
#> Paused at: Step2

4. Inspecting the Checkpoint

Even though the execution paused, the thread_id and the state (including Step1’s result) are safely stored in DuckDB.

con <- DBI::dbConnect(duckdb::duckdb(), db_path)
DBI::dbGetQuery(con, "SELECT thread_id, updated_at FROM agent_checkpoints")
#>            thread_id          updated_at
#> 1 reprex-session-001 2026-04-01 13:43:03
DBI::dbDisconnect(con, shutdown = TRUE)

5. The Restart (The Success)

Now we “fix” the state and restart from Step2. HydraR detects the existing checkpoint, merges our fix (fixed = TRUE) into the restored state, and re-executes from Step2 onwards — skipping Step1 entirely.

# Fixing the state: Set 'fixed' to TRUE
new_state <- list(fixed = TRUE)

# Run 2: Resume from Step2 using the SAME thread_id
final_results <- dag$run(
  thread_id = tid,
  checkpointer = saver,
  initial_state = new_state,
  resume_from = "Step2"
)
#> Graph compiled successfully.
#> [Iteration] Restored state from checkpoint for thread: reprex-session-001
#> [Resuming] Linear DAG Execution from node: Step2
#> [Linear] Running Node: Step2
#>    [Step2] Executing R logic...
#> [Linear] Running Node: Step3
#>    [Step3] Executing R logic...

# Verify results
cat("Final Status:", final_results$status, "\n")
#> Final Status: completed
cat("Step2 Output:", final_results$results$Step2$output, "\n")
#> Step2 Output: System recovered!
cat("Step3 Output:", final_results$results$Step3$output, "\n")
#> Step3 Output: All steps finished.

Summary

The DuckDB checkpointer ensures that no computational progress is lost. This is critical for agentic workflows involving expensive LLM calls or long-running bioinformatics pipelines.
