Advanced Network Logic Circuitry

HydraR supports complex “Network Logic Circuitry” that goes beyond simple Directed Acyclic Graphs. This includes dynamic routing, self-healing networks via error edges, and parallel isolation.

Core Concepts

  1. Dynamic Routing: Use a Router node to decide which path to follow at runtime based on the current state.
  2. Error Handling: Define error_edges to automatically switch to recovery nodes if a task fails.
  3. Declarative Control: Define these complex relationships in YAML, keeping your R code strictly focused on high-level orchestration.

Loading the Circuit

Running the Scenario

In this scenario, we set simulate_fail = TRUE to witness the self-healing capability.

# Run the DAG with 'fast' mode and 'simulate_fail = TRUE'
final <- dag$run(wf$initial_state)

Resilience Features

  • Router Nodes: Allow for branching that is decided by logic, not just hardcoded paths.
  • Error Edges: Ensure that critical failures don’t crash the entire workflow but instead trigger cleanup or recovery routines.
  • State Isolation: Combined with use_worktrees = TRUE, these branches can run in complete isolation from one another.
