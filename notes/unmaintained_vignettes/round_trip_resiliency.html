Round-Trip Resiliency: Status-Colored Plotting

This vignette demonstrates how to use the HydraR status-colored plotting API. By defining a workflow in YAML and running it, you can generate a status-colored Mermaid visualization that highlights nodes based on their execution outcome.

The Quality Control Loop

We simulate a “Quality Control” loop where initial findings from a search are checked. If the check fails, the graph routes execution back to a “Deep Search” node to gather more detail before re-summarizing.

library(HydraR)

# Load the resilient loop from YAML
wf <- load_workflow("round_trip_resiliency.yml")

# Spawn and compile the DAG
dag <- spawn_dag(wf)

Running the Scenario

In this example, the Check Quality node is hardcoded to fail on its first run and succeed on its second.

# Run with a max_steps limit to prevent infinite loops
final <- dag$run(wf$initial_state, max_steps = 10)

Generating the Status Plot

After the execution, the plot(status = TRUE) method uses the internal trace_log to apply styles to the Mermaid graph: - Green: Successful execution. - Red: Failed execution. - Yellow: Paused or currently active.

# Export status-colored Mermaid
dag$plot(status = TRUE)
#> ```mermaid

Summary

HydraR ensures that even complex, cyclic graphs can be audited and visualized effectively, making it easier to debug multi-agent coordination issues and trace the path taken through the consensus engine.
