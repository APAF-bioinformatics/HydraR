LLM Fan-Out and Fan-In Synthesis

This vignette demonstrates the Fan-Out / Fan-In pattern using HydraR. Instead of manual node configuration, we use a declarative YAML syntax to define complex multi-agent topologies.

Collaborative Storytelling Pattern

We model a collaborative writing process where a Director sets the stage, multiple specialized Writers (Action, Mystery, Romance) independently generate plot lines in parallel, and a Master Editor synthesizes them into a cohesive story.

This pattern is ideal for: 1. Parallel Creativity: Generating diverse perspectives on the same input. 2. Synthesis: Combining isolated agent outputs into a final consensus.

Declarative Orchestration

We define the architecture in parallel_fan_out.yml.

library(HydraR)

# Load the workflow from YAML
wf <- load_workflow("parallel_fan_out.yml")

# Spawn and compile the DAG
dag <- spawn_dag(wf)

Running the Parallel Chain

When dag$run() is called, the Director node executes first. Then, the Three writer nodes are evaluated (potentially in parallel if furrr is configured). Finally, the Master Editor consolidates the results.

# Execute with the initial premise
final <- dag$run(wf$initial_state)

# View the final synthesized story
cat(final$state$get("editor"))

Scaling with Worktrees

For heavy parallel processing (e.g., each writer needing its own git environment), you can set use_worktrees = TRUE in the run() method. This ensures that each “branch” of the Fan-Out has a completely isolated filesystem for execution, preventing race conditions.
